# HarpHymnal Drills — tablet app

An Android WebView flash-card app for solo lever-harp practice. Two drill
categories:

1. **Pool drill** — all 118 fractions in the HarpHymnal pool
   (paths + reserve). One card per fraction.
2. **Technique drill** — for each of the 14+ technique sections in
   `SDD.md` §3.6.1, cycle through every distinct from→to gesture pair.

Tap the right half of a card (or swipe left / press Next / Space) to
advance. Tap the left edge (or swipe right / press Prev) to go back.
"Home" at the top-left returns to the tile grid.

Mirrors the pattern of `../../trefoil/app/` — single Activity that loads
`file:///android_asset/index.html` in a full-screen WebView with
`KEEP_SCREEN_ON`.

## 1. (Re)generate the drill data

```bash
python3 tablet_app/build_drill_data.py
```

Run from the repo root. Writes
`tablet_app/app/src/main/assets/harphymnal_drills.json` containing:

```jsonc
{
  "schema_version": 1,
  "source": "HarpTrefoil.json + SDD.md §3.6.1",
  "filters": { "MIN_GAP": 0, "MAX_GAP": 6, "note": "…" },
  "pool_cards":       [ /* 118 entries */ ],
  "technique_drills": { /* 19 keys: Third sub, Quality sub, … */ },
  "summary": {
    "pool_card_count": 118,
    "technique_count": 19,
    "technique_card_total": 536
  }
}
```

Sources read: `data/trefoil/HarpTrefoil.json` (the pool) and `SDD.md`
(§3.6.1 technique inventory). The SDD inventory already has the three
HarpHymnal drill filters applied:

1. Duplicate pattern-pair dedup (LH-pattern, RH-pattern identity).
2. Same-string overlap reject (`gap >= MIN_GAP = 0`).
3. Wide-spread reject (`gap <= MAX_GAP = 6`).

Bounds come from `grammar/constants.py` (imported for the record).

## 2. Build the APK

```bash
cd tablet_app
./gradlew assembleDebug
```

Output: `app/build/outputs/apk/debug/app-debug.apk`.

The Android SDK must be on `ANDROID_HOME` and a JDK 17 on `JAVA_HOME`
— same requirements as `../trefoil/app/`. For example:

```bash
ANDROID_HOME=$HOME/Android/Sdk \
JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64 \
./gradlew assembleDebug
```

## 3. Install on the tablet

Tablet setup (once):

1. **Settings → About tablet** → tap *Build number* seven times to unlock
   Developer Options.
2. **Settings → Developer Options** → enable *USB debugging*.
3. Plug the tablet into USB. When the "Allow USB debugging?" prompt shows
   on the tablet, accept.

Verify the tablet is visible to `adb`:

```bash
adb devices
# List of devices attached
# ABCD1234   device
```

Then install:

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

The app appears on the tablet as **HarpHymnal Drills**. Launch it; the
first screen is a tile grid of Pool + 19 technique tiles. Grey tiles are
techniques the SDD currently lists with no concrete pattern-pair examples
(e.g. *Modal reframing*, *Voice leading*, *Anticipation*, *Delay*).

## 4. Quick desktop preview (no SDK needed)

The WebView side is pure HTML + CSS + JS; you can preview it in any
browser. `fetch` of the sibling JSON fails under `file://` on most
browsers, so run a tiny local server:

```bash
cd tablet_app/app/src/main/assets
python3 -m http.server 8000
# visit http://localhost:8000/index.html
```

## Scope (what is NOT included, by design)

- No audio playback, metronome, MIDI, or scoring.
- No note-rendering engine; flash cards use the SDD's literal
  `lowercase=LH UPPERCASE=RH` letter strings (key of C).
- No external dependencies — no AndroidX, no Jetpack, no frameworks on
  the HTML side.
