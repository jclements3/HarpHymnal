Encoding-system rewrite — in progress this session, not yet committed

The repo-root encoding spec has been rewritten end-to-end today. The previous README described a chord-naming system (Roman numeral chords

  • mode/intervals + handout chord labels). The current README describes a fundamentally different system: a shape-setup encoding that catalogs hand placements on the strings and progressions of those placements, explicitly not music notation.

Key reframe (do not lose this): the system encodes what is committed to in the air before the hand touches the strings. Rhythm, articulation, strike order after contact, timing, and dynamics are deliberately outside scope. Most of what looks like “playing notation” (operators, hand markers, space-separated shape sequences) is actually about sequences of pre-contact configurations, not sequences of played notes. There’s a project-memory entry under memory/project_encoding_scope.md — read that first if you didn’t participate in today’s session.

What changed in the file system today (uncommitted):

  • README.md — full rewrite. Three octave-prefix forms: absolute (1x, 2L, 7R — explicit harp octave 1–7 plus letter slot for hand), relative (+1x, -2L — sign required, prior-octave plus/minus N), and absent (sticky inheritance from prior shape; drill context may leave it unstated entirely). Disambiguator is the first character class. Subscripts now mean assembly order (which finger plants 1st–8th in the assembly sequence; ₁–₄ for one-hand shapes, ₁–₈ global for two-hand), not finger labels — the previous README had this wrong. Hand assignment lives in the letter slot of the octave prefix (x = either, L = left, R = right): 1L6̂ is a left-hand shape, 1R1̂333 is right-hand. Two-hand shapes (≥ 4 intervals) use a comma inside one shape to split LH and RH intervals (1x1̂333,3333). Shapes within a row are space-separated. Header block: Title: (optional) + Key: (required), separated from rows by ---. No Meter: — rhythm is out of scope.
  • VERIFY.md — reach-calibration drills. User has not run these yet; when they do, the per-finger reach ceilings in the README’s table will be replaced with calibrated numbers.
  • DRILLS.md — full replacement. Old chord-pool letter notation (cegDFA etc.) is gone. Drills under the new grammar, including the canonical thirds chain (1x1̂333$~), modal cycle, two-hand bass+chord progressions (1L1̂ 3R1̂333 …), and a top-down assembly study using subscripts (1x1̂₄3₃3₂3₁).
  • HANDOUT.md — new file. Quick-reference catalog of the 12 universal-winner shape patterns plus mode-conditional gems, common setup forms (single notes, dyads, triads, tetrads), and sequence patterns. Different in purpose from the older handout.md/handout.tex which use the previous chord-name system; those are unchanged.
